# Decision Record

- **Date:** 2026-03-16
- **Decision:** Tighten ClawText publication framing to be evidence-disciplined rather than maximally aggressive
- **Context:** After release-truth, retrieval, isolation, and continuity hardening passes, the publication brief remained directionally strong but overstated certainty in a few places, especially quantified impact ranges and the implied strength of end-to-end proof.
- **Why this choice:**
  - strong publication copy is still compatible with honest uncertainty boundaries
  - avoiding overclaiming now reduces future cleanup and protects trust
  - README presentation can be improved later without requiring factual rollback
- **What changed:**
  - POST_BRIEF status and launch language were tightened
  - directional metrics were reframed as directional until stronger evidence exists
  - release notes now explicitly preserve current proof-gap boundaries
- **Follow-up required:**
  - revisit README presentation after retrofit/hardening is complete
  - strengthen claims later only if stronger proof artifacts are captured
